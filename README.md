## 🌟 项目简介

本系统是一个面向真实临床场景的先进医疗多智能体系统（Multi-Agent System）。有别于传统的单一对话机器人，本项目在架构上采用了业界最前沿的 **图计算工作流（LangGraph）** 与 **全分布式微服务（基于 Model Context Protocol, MCP）**，旨在解决医疗大模型在实际应用中普遍面临的“幻觉横行”、“流程不严谨”以及“重型影像算力资源难以跨端部署”的核心痛点。

本系统以 **DeerFlow** 为轻量级调度中枢（大脑），将负责医学影像分析的重型 PyTorch 算法库和 RAG 知识检索库剥离为独立的远端服务（手脚），实现了一套具备 **多期并发检查（Fan-out）、智能会诊流转** 以及 **强制医疗安全质控拦截（Guardrails）** 的企业级自动临床流水线。

## 🏗️ 核心架构说明 (架构亮点)

为了保证最极致的工程可靠性与环境隔离，本项目的源码采取了 **Monorepo（单体多模块仓）** 的先进业务组织形式。

项目根目录下拆分了三个绝对物理隔离的核心微服务：

### 1. `1_core_orchestrator`（核心调度大脑）
- **技术栈**：Python, LangGraph, DeerFlow 架构
- **作用**：纯 CPU 运行的极轻量级“住院总医师”。内置了 A0（智能分发）、A1（导诊护士）、A2（检查专员）、A3（主治医师）等全套 Agent 的 Prompt 及协作状态机。
- **独创亮点**：内置了基于 Guardrail 的 A5（医疗质控系统），作为执行图的最后一道防线，利用严格规则拦截“擅自确诊”、“擅自开药”等高危大模型输出幻觉。

### 2. `2_mcp_ragflow_lite`（私有领域知识库）
- **技术栈**：RAGFlow, 向量数据库
- **作用**：基于 MCP 协议独立运行的知识外挂。负责提供最新的临床指南检索与循证医学证据支撑（Evidence-based Medicine）。

### 3. `3_mcp_medical_vision`（重型医疗影像算法集群）
- **技术栈**：PyTorch, DenseNet, CUDA, mcp-sdk
- **作用**：医疗领域专用的视觉 AI 算力节点。将极重度的医学影像分析（如 X 光病灶分割）、化验单 OCR 解析全部封装为远端的工具函数。
- **架构收益**：通过 MCP 实现了**彻底的算力解耦**。核心大脑无需安装任何几 GB 大小的深度学习库，解决了传统 AI 应用依赖地狱的终极痛点。

## 🚀 部署与运行指南

由于项目采用微服务解耦设计，我们在答辩及测试时，可以**仅运行轻量级的中枢神经（模块 1），而将较重的模型节点（模块 2/3）按需接入**。

### Step 1: 启动影像与检索节点 (可选，需 GPU 机器)
1. 进入 `3_mcp_medical_vision` 目录。
2. 安装其专属的重型依赖：`pip install -r requirements_gpu.txt`（如 PyTorch 等）。
3. 启动 MCP 挂载服务：`python vision_mcp_server.py`
*(如果您是审核评委，且本电脑无 GPU，也可跳过此步骤，调度大脑仍会正常接管导诊及无图逻辑。)*

### Step 2: 启动核心调度大脑 (必需)
1. 进入 `1_core_orchestrator` 目录。
2. 安装轻量级环境：`pip install -r requirements.txt` (仅需 Langchain 等基础网络库)
3. 配置环境：复制 `.env.example` 为 `.env`，填入对应的大模型 API 密钥（本项目适配且极力推荐配置 `DeepSeek V3` 及 `Qwen-VL-Max`）。
4. （自动挂载）：系统会根据 `extensions_config.json` 自动扫描并在本地或远程映射挂载上面的外置算法模块。
5. 运行节点服务启动网关。

## 💡 为什么这样设计？(致评委)

我们深知医疗 AI 容不得半点沙子。如果使用简单的 `While` 循环堆砌多智能体，不仅状态难以回溯，且在医院的真实网络环境下极易导致进程崩溃死锁。

因此，我们的心血 100% 投入在了对 **原生图引擎（Graph Engine）时间轴回溯追踪技术** 的应用，以及对 **最新的人机/机机异步通信协议（MCP）** 的前瞻性落地中。不仅解决了一个医疗长流程会诊的需求，更为后续接入更多如“心电图分析仪”、“基因测序库”奠定了教科书级的可拓展工程地基。
