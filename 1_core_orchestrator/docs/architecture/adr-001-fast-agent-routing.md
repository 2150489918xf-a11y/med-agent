# ADR-001: 子 Agent 调用极致速度优化（6 层火箭方案）

## 状态

拟议中 (Proposed)

## 背景

用户要求对影像分析流程进行**极致速度优化**。当前从用户上传图片到看到结果需要约 **2 分钟**，目标是压缩到 **10 秒级别**。

## 延迟拆解（当前 ~120 秒）

```
用户发送消息 ──[10s]──▶ 主Agent思考并分发
   ──[10s]──▶ 子Agent(qwen3-vl-235b)首次LLM推理
   ──[60-90s]──▶ MCP引擎加载4个模型+推理
   ──[10s]──▶ 子Agent二次LLM推理(组装报告)
   ──[2s]──▶ 结果返回主Agent
```

## 6 层极速优化方案

### 🚀 第 1 层：模型常驻内存（消灭 60-90 秒）

**问题**：`engine.py` 每次调用都从磁盘重新加载 YOLO(52MB)、PSPNet、DenseNet、MedSAM(350MB)。

**方案**：用 `@lru_cache` 或模块级全局变量实现惰性加载单例。模型在首次推理时加载，后续直接复用。同时在 `server.py` 启动时调用 `warmup()` 预热。

**预期收益**：推理阶段从 60-90 秒 → **3-8 秒**。

---

### ⚡ 第 2 层：推理管线并行化（再省 2-3 秒）

**问题**：`engine.py` 中 YOLO 检测和 DenseNet 分类是**完全独立的**，但目前串行执行。

**方案**：用 `concurrent.futures.ThreadPoolExecutor` 将 `phase1_inference`（YOLO+PSPNet）和 `get_densenet_probs` 并行执行。两者共享 GPU 锁但可交替占用。

**预期收益**：省 **2-3 秒**。

---

### 🎯 第 3 层：MedSAM 按需禁用（省 3-5 秒）

**问题**：MedSAM 精细分割只对实心病灶有意义，但即使没有实心病灶也要加载 350MB 权重。

**方案**：默认传 `enable_sam=False`。前端审核面板可以手动触发精细分割（后续增强功能）。

**预期收益**：省 **3-5 秒**（跳过整个 Phase 3）。

---

### 🧠 第 4 层：子 Agent 大幅轻量化（省 20-30 秒）

**问题**：`imaging-agent` 使用 `qwen3-vl-235b`（2350 亿参数超大视觉模型），但它根本**不需要看图**！它只是个"快递员"——收到路径，转发给 MCP，格式化结果。

**方案**：将默认模型改为 `qwen-plus`（或同等轻量模型）。两次 LLM 调用合计从 20-30 秒 → **3-5 秒**。

**预期收益**：省 **20-30 秒**。

---

### ⚡ 第 5 层：闪电路由中间件（省 5-10 秒）

**问题**：即使主 Agent 不做深度思考，它生成 `task` Tool Call 仍需一次完整的 LLM 推理（~5-10 秒）。

**方案**：在主 Agent 的中间件链路前端插入 `FastRouteMiddleware`。检测到用户刚上传了图片（通过 OpenCV/Chinese-CLIP 的 `.meta.json` 确认为 `medical_imaging` 类型）时，直接伪造 `task(subagent_type="imaging-agent")` 调用，完全跳过主模型的推理。

**预期收益**：省 **5-10 秒**。

---

### 🔧 第 6 层：子 Agent 执行器轻量化（省 1-2 秒）

**问题**：`SubagentExecutor` 在每次执行时：
- 创建全新的 `asyncio` 事件循环（`asyncio.run()`）
- 重新从配置中加载全部 MCP 工具（虽有缓存但仍有解析开销）
- 5 秒轮询间隔过长

**方案**：
- 将 `task_tool` 的轮询间隔从 5 秒降为 1 秒
- 将 `max_turns` 从 30 降为 10（影像 Agent 只需 2-3 轮）

**预期收益**：省 **1-5 秒**。

---

## 优化效果预估

| 层级 | 优化项 | 当前耗时 | 优化后 | 节省 |
|---|---|---|---|---|
| 1 | 模型常驻 | 60-90s | 3-8s | **57-82s** |
| 2 | 推理并行 | 含在上面 | -2s | **2-3s** |
| 3 | MedSAM 跳过 | 含在上面 | -3s | **3-5s** |
| 4 | 子 Agent 轻量化 | 20-30s | 3-5s | **17-25s** |
| 5 | 闪电路由 | 5-10s | 0s | **5-10s** |
| 6 | 执行器调优 | 5-10s | 1-2s | **3-8s** |
| **合计** | | **~120s** | **~10-15s** | **~105-115s** |

> [!TIP]
> 首次分析（冷启动，模型尚未加载）约 30-40 秒。**第二次** 及之后的分析预计 **10-15 秒** 即可出结果。
